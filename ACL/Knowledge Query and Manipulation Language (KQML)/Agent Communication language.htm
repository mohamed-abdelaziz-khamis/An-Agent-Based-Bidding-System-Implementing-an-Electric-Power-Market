Agent Communication language (ACL)

 

Software agents need to be able to interact and to communicate with other agents, and their environment, in order to cooperate and to share knowledge. To do so, requires that they have a common or inter-translatable representation language and to share a framework of knowledge to interpret the exchanged messages. There is no universally accepted language in which to represent information and queries. However, a language those agents might use for these interactions should have a number of desirable features. Finin and Labrou have come up with these desiderata:

 

Form: A good agent communication language should be declarative, syntactically simple and readable by people.

 

Content: A communication language should be layered in a way that fits well with other systems. In particular, a distinction should be made between the communication language, which expresses communication acts, and the content language, which expresses information about the domain.

 

Semantics: The semantics of a communication language should exhibit those properties expected of the semantics of any other language. It should be grounded in theory, and also ambiguous.

 

Implementation: The implementation should be efficient, both for speed and for bandwidth utilisation.

 

Networking: An ACL should fit well with modern networking technology.

 

Environment: The environment should be highly distributed, heterogeneous, and very dynamic.

 

Reliability: A communication language must support reliable and secure communications.

 

The implementation language, agent system architecture and the intended application will also influence the design of an ACL. As agent applications become more complex, the need arises to express and maintain more knowledge about its environment, history, and its goal, because an agent should react autonomously on events in its environment, it needs a model of the environment and the set of rules that governs its behaviour. Hence, an ACL should be a knowledge representation language that permits exchange of knowledge between agents.

 

To establish communication between agents, there is a need for a common frame of reference, or the context of the message. The context of the message determines how that particular message is interpreted. Hence, the need to conceptualise the world using ontology.

 

Ontology

 

The word ontology has a long history in philosophy, where it refers to the notion of existence. In the context of ACL, ontology is a specification of a conceptualisation. Conceptualisation is sets of concepts and the relationships that hold between these sets.

 

Ontology is used to define the vocabulary that agents must use to maintain knowledge and the behavioural rules. If ontology is shared amongst agents in an environment, then the vocabulary can be used for communication between these agents. Therefore, ontology is written using a knowledge representation language.

 

Knowledge Representation Language

 

In order for an agent to use and maintain knowledge about its environment, history, and its goal, a knowledge representation language is used. The Knowledge Sharing Effort, KSE, “is a consortium to develop conventions facilitating sharing and reuse of knowledge bases and knowledge-based systems.”

 

The KSE is interested in common knowledge base languages, common constructs among languages, shared and reusable knowledge bases, and the external interfaces (a high level interaction language).

 

KSE has developed the Knowledge Interchange Format, KIF. KIF is used to express simple facts as well as more complex relations through a number of logical operators. In addition, KIF provides constructs for describing procedures. KIF is first order predicate logic. It is a simple list-base linear ASCII syntax.

To facilitate successful interoperation between agents, there is a need for standardised means and methods of the exchanged messages. Hence, a communication protocol is required.

 

Communication Protocols

 

KIF is only a common representation language. There is a need for a language for agents to share a framework of knowledge to interpret the exchanged message. There are two protocols that have dominated the debates and discussion amongst researchers, practitioners, and standard bodies. They are FIPA-ACL and KQML.

 

FIPA, the Foundation for Intelligent Physical Agents, has put forward an agent communication language, FIPA-ACL, which is based on the Speech Act Theory. FIPA-ACL assumes existence of an Agent Management system, not part of the language, and abstracts away the low-level communication details.

 

The Knowledge Query and Manipulation Language, KQML, was developed as part of the KSE. It is by far the most widely used in existing systems, and thus, shall be examined further in this article.

 

Knowledge Query and Manipulation Language (KQML)

 

Both FIPA-ACL and KQML provides primitives to allow agents to communicate the nature of their request and to find the other agent suitable to deal with that request. However, in KQML, the primitives are referred to as performatives.

 

KQML is designed to achieve tasks at three conceptual layers:

 

 

Figure2: 3 Layers Of KQML

 

 

Distinctions between layers are not explicit in programs and specifications. This makes building good abstractions difficult.

 

Performatives in KQML is like instructions. They can be divided in to several categories.

 

 

 

Queries

These performatives are used to send questions for evaluation somewhere.

Responses

Used by agents in order to reply to queries and requests.

Informational

Informational performatives are used to transfer information.

Generative

This class of performatives is used for controlling and initiating the exchange of messages.

Capability definition

Allows agents to learn about the capability of other agents and to announce its own to the agent community

Networking

Networking performatives make it possible to pass directives to underlying community layer.

The performatives define the permissible operations that agents may attempt to access each other knowledge or goal stores. These performatives are the foundation on which the higher-level inter-agents interaction concepts such as negotiations and contract nets are based. In addition, KQML provides a basic architecture for knowledge sharing through a special class of agents called the community facilitators, which organise and coordinate the interactions of other agents.

 

Conclusion

 

The need for an Agent Communication Language is clear. It is vital in the advancement of electronic commerce, as well as other applications, as the Internet will heavily depend on the inter-operability between agent systems. There are many protocols in existence and in development, but the need for a convergence towards a single standard format is of equal importance to, if not more important than, the improvement in agent technology.
